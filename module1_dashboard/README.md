# Module 1: 心血管疾病数据可视化仪表板

## 功能概述

这是 CardioAI 系统的第一个模块，提供交互式的数据可视化和探索功能。

### 核心功能

✅ **数据加载与缓存**
- 自动加载心血管疾病数据（70,000 条记录）
- 使用 `@st.cache_data` 优化性能
- 首次加载后缓存，后续访问秒开

✅ **特征工程**
- 年龄转换：天数 → 年龄（四舍五入）
- BMI 计算：weight / (height/100)²
- BMI 分类：偏瘦、正常、超重、肥胖（中国标准）
- 类别转换：胆固醇、血糖描述性标签

✅ **数据清洗**
- 删除舒张压 ≥ 收缩压的记录
- 收缩压范围：[90, 250] mmHg
- 舒张压范围：[60, 150] mmHg
- 清洗率约 10-15%

✅ **交互式筛选**
- 年龄范围滑块（动态范围）
- 性别多选框（女性/男性）
- 疾病状态多选框（有/无）
- 实时更新所有图表和指标

✅ **关键指标展示**
- 筛选后记录数
- 心血管疾病风险率
- 数据清洗统计
- 平均年龄
- 平均 BMI

✅ **可视化图表**
1. **年龄分布直方图**：观察不同年龄段的疾病分布
2. **BMI 分类堆叠图**：分析体重指数与疾病的关系

✅ **数据概览表格**
- 显示前 100 条筛选后的数据
- 支持排序和搜索
- 友好的列名显示

## 运行方式

### 方法 1: 命令行启动

```bash

conda activate cardioenv

# 2. 进入模块目录
cd D:\cursor_pro\pro1\module1_dashboard

# 3. 运行 Streamlit 应用

```

### 方法 2: 指定端口启动

```bash
streamlit run cardio_dashboard.py --server.port=8501
```

### 方法 3: 后台运行（无浏览器）

```bash
streamlit run cardio_dashboard.py --server.headless=true
```

## 数据说明

### 数据源
- 文件路径: `D:/cursor_pro/pro1/data/心血管疾病.xlsx`
- 数据量: 70,000 条记录
- 数据字段: 13 个特征

### 字段说明

| 字段 | 说明 | 单位/范围 |
|-----|------|----------|
| id | 患者ID | 整数 |
| age | 年龄 | 天（原始）|
| age_years | 年龄 | 年（转换后）|
| gender | 性别 | 1=女性, 2=男性 |
| height | 身高 | cm |
| weight | 体重 | kg |
| bmi | BMI | 计算值 |
| bmi_category | BMI分类 | 偏瘦/正常/超重/肥胖 |
| ap_hi | 收缩压 | mmHg |
| ap_lo | 舒张压 | mmHg |
| cholesterol | 胆固醇 | 1=正常, 2=高于正常, 3=远高于正常 |
| gluc | 血糖 | 1=正常, 2=高于正常, 3=远高于正常 |
| smoke | 吸烟 | 0=否, 1=是 |
| alco | 饮酒 | 0=否, 1=是 |
| active | 运动 | 0=否, 1=是 |
| cardio | 心血管疾病 | 0=无, 1=有 |

## BMI 分类标准（中国标准）

| 分类 | BMI 范围 |
|-----|---------|
| 偏瘦 | < 18.5 |
| 正常 | 18.5 - 24 |
| 超重 | 24 - 28 |
| 肥胖 | ≥ 28 |

## 常见问题

### Q: 数据文件找不到？
A: 确保数据文件在 `D:/cursor_pro/pro1/data/心血管疾病.xlsx`，路径必须正确。

### Q: 筛选后没有数据？
A: 检查筛选条件是否过于严格，尝试放宽年龄范围或选择更多选项。

### Q: 图表显示缓慢？
A: 首次加载需要处理 70,000 条数据，后续访问会快很多（已缓存）。

### Q: 如何停止应用？
A: 在终端按 `Ctrl+C` 停止 Streamlit 服务。
