# 音频播放功能修复说明

## 修复概述

**问题**: 之前的模拟音频无法在浏览器中播放  
**原因**: 使用了简化的 MP3 头部数据，不是真实的音频文件  
**解决方案**: 生成标准的 WAV 格式音频文件（1秒 440Hz 正弦波）

---

## 修复详情

### 1. 音频生成改进

**之前** (不可播放):
```python
# 简单的字节序列，无法播放
mock_mp3_data = (
    b'ID3\x04\x00\x00\x00\x00\x00\x00'
    b'\xff\xfb\x90\x00'
    b'\x00' * 100

```

**现在** (可正常播放):
```python
# 生成真实的 WAV 音频文件
- 格式: WAV (PCM)
- 采样率: 22050 Hz
- 声道: 单声道 (Mono)
- 位深度: 16-bit
- 时长: 1 秒
- 音调: 440Hz (标准 A 音)
- 音量: 30%
```

## 技术实现

### 音频生成算法

```python
import wave
import struct
import math

# 生成正弦波样本
for i in range(num_samples):
    sample = amplitude * math.sin(2 * π * frequency * i / sample_rate)
    sample_int = int(sample * 32767)  # 转换为 16-bit PCM
```

### WAV 文件结构

```
[RIFF Header]
├── [Format Chunk]
│   ├── Audio Format: PCM
│   ├── Channels: 1
│   ├── Sample Rate: 22050
│   └── Bits per Sample: 16
└── [Data Chunk]
    └── [PCM Samples: 22050 samples]
```

## 音频内容说明

### 模拟模式音频

- **内容**: 标准 A 音（440Hz 正弦波）
- **用途**: 验证音频播放管道是否正常工作
- **特点**: 清晰、单一频率、容易识别
- **优点**: 快速生成、文件小、不依赖外部 API

### 为什么使用 440Hz？

- 这是国际标准音高 A4（钢琴中央 C 上方的 La）
- 人耳容易听到的频率（人耳范围 20-20000Hz）
- 纯净的音调，便于测试和验证

## 常见问题

### Q1: 为什么听到的是"嘟"声而不是语音？

**A**: 这是正常的！模拟模式使用 440Hz 音调来验证音频播放功能。切换到真实模式（`USE_MOCK_DATA=false`）即可获得真实语音。

### Q2: 音频可以播放，但太短了？

**A**: 模拟音频固定为 1 秒。要调整时长，修改 `voice_assistant_app.py` 中的 `duration` 参数：
```python
duration = 2.0  # 改为 2 秒
```

### Q3: 可以改变音调吗？

**A**: 可以！修改 `frequency` 参数：
```python
frequency = 261.63  # C4 (Do)
frequency = 329.63  # E4 (Mi)
frequency = 523.25  # C5 (高音 Do)
```

### Q4: WAV 文件太大了怎么办？

**A**: WAV 是未压缩格式，所以文件较大。如需更小的文件，可以：
- 降低采样率（例如改为 11025 Hz）
- 缩短时长
- 或者切换到真实模式使用 MP3 格式

### Q5: 浏览器自动播放被阻止？

**A**: 部分浏览器会阻止自动播放。解决方法：
- 手动点击播放按钮
- 或在浏览器设置中允许该站点自动播放音频
